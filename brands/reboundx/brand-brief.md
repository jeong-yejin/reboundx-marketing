# 브랜드명
ReboundX

## 서비스 카테고리
crypto fee rebate & referral payback platform
- 사용자가 외부 거래소(CEX + Perp DEX)에서 거래하면 발생하는 수수료 일부를 ReboundX를 통해 돌려받는 구조
- 거래를 직접 실행하지 않음. 거래소와 사용자 사이에서 리베이트 구조를 중개
- CEX와 Perp DEX를 아우르는 업계 최고 수준의 리베이트율을 표방

## 공식 태그라인
"The highest-rate referral rebate platform across top CEXs and Perp DEXs, ensuring traders get more back from every transaction."
"Transform trading fees from a cost into an opportunity."

## 핵심 가치 제안
- 이미 내고 있는 거래 수수료를 돌려받을 수 있음
- 별도의 트레이딩 전략 변경 없이 기존 거래 습관 유지하면서 수익 발생
- 24시간 이내 자동 정산 (payout SLA)
- 온체인 + 오프체인 정산 내역 완전 투명 공개
- CEX + Perp DEX 단일 대시보드 통합 관리

## 플랫폼 작동 방식 (사용자 플로우)
1. ReboundX 레퍼럴 코드/링크를 통해 파트너 거래소에 가입
   - 주의: ReboundX 코드 없이 가입하면 UID 연결 불가 → 리베이트 수령 불가
2. 거래소에서 KYC 완료 (ReboundX 가이드 제공)
3. UID(User ID), web3 지갑 주소를 ReboundX 계정에 연결
4. 기존 방식대로 거래
5. 수수료 자동 정산 → 24시간 이내 지급

## 주요 기능 (사이트 기준)

### 1. Fee Rebate Collection
- 파트너 거래소 거래 수수료 일부를 자동으로 정산 및 지급
- 온체인·오프체인 정산 내역 투명 공개

### 2. Signup / KYC / UID Linking Guides
- 각 파트너 거래소별 가입 → KYC → UID 연결 단계별 가이드
- URL 구조: `/en/guide/signup/[exchange]`

### 3. Partner Exchange Guide (`/en/exchanges`)
- 파트너 거래소 전체 목록 및 리베이트 혜택 비교
- 거래소별 상세 페이지 (예: `/en/exchanges/grvt`)

### 4. Perpdex Comparison & Analysis Tool (`/en/perpdex`)
- 주요 Perp DEX 거래소(Hyperliquid, dYdX, edgeX 등) 비교
- 실시간 거래량 모니터링
- 에어드랍 수익 시뮬레이터 (airdrop calculator)

### 5. Trading Competitions
- 파트너 거래소와 협업한 트레이딩 컴피티션 (예: Backpack Trading Competition)
- 상금 풀, 참가 조건, 순위 보상 포함

- 텔레그램 커뮤니티 의존도 매우 높음

## 타겟 사용자

### Primary — 수수료 구조를 이해하는 중급 트레이더
- 거래소를 1개 이상 사용 중 (CEX 또는 Perp DEX)
- 월 거래량이 일정 수준 이상 (수수료 부담을 인식함)
- 리베이트 개념은 알지만 어디서 받는지 몰랐거나 귀찮아서 안 함
- 트위터(X), 텔레그램 활동
- Perp DEX 트레이더 포함 (신규 세그먼트)

### Secondary — 크립토 입문자 / 거래소 가입 준비 중인 사용자
- 거래소를 처음 선택하는 단계
- 수수료 개념 자체가 생소함
- ReboundX 링크로 거래소 가입 시 리베이트 구조로 편입 가능
- 유튜브, 블로그, SNS로 정보 탐색 중
- 주의: ReboundX 링크 없이 가입 시 UID 연결 불가 → 온보딩 단계에서 레퍼럴 링크 사용 필수 안내

## 사용자 상태 4가지 (UX 기준)
1. 페이백이 뭔지 모르는 사용자 → 구조 설명이 먼저
2. 연결했지만 혜택을 체감하지 못하는 사용자 → 수치 가시화가 먼저
3. 정산과 출금 구조를 신뢰하지 못하는 사용자 → 투명한 정산 흐름 노출이 먼저
4. ReboundX 링크 없이 거래소에 이미 가입한 사용자 → UID 연결 불가 안내 + 재가입 유도

## 지역 타겟

### 미국
- 규제 환경에 민감. 리베이트를 "수익"이 아닌 "수수료 절감"으로 포지셔닝해야 함
- 헤비 트레이더 비중 높음. 거래량 기반 리베이트 수치에 반응
- 주요 접점: 트위터(X), 레딧(r/CryptoCurrency, r/ethfinance)
- 신뢰 지표 중시. 정산 투명성과 온체인 검증 가능 여부에 관심
- 영어 콘텐츠 퀄리티와 공식성이 신뢰에 직접 영향

### 동남아 (베트남, 태국, 인도네시아, 필리핀)
- 크립토 침투율 높고 리테일 트레이더 비중 큼
- 수수료 절감에 대한 민감도 높음. 소액 리베이트도 체감 효과 큼
- 커뮤니티 기반 신뢰 형성. 텔레그램 로컬 그룹이 핵심 채널
- KOL(인플루언서) 레퍼럴 구조가 유입에 효과적
- 모바일 중심 사용 환경. 웹 UI는 모바일 최적화 필수
- 언어별 현지화 필요: 베트남어, 태국어, bahasa Indonesia, Filipino

### 공통 유의사항
- 지역별 규제 차이 존재. "투자 수익" 표현 전면 회피
- 미국: SEC 관련 민감도 높음. 리베이트를 fee cashback/rebate으로 명확히 표현
- 동남아: 사기 플랫폼에 대한 경계심 높음. 신뢰 구축 콘텐츠 우선

## 운영 채널
- 트위터(X): 주요 크립토 유저 접점. 캠페인 및 리베이트 인증 콘텐츠
- 텔레그램: 커뮤니티 운영, 공지, 정산 안내
- 웹사이트: 온보딩, 거래소 연결, 대시보드, 정산 확인
- 다국어 지원: 영어 + 한국어 (추가 현지화 예정)

  - DJ 애프터파티

## 경쟁 포지셔닝
- CEX + Perp DEX 아우르는 가장 높은 수준의 리베이트율 표방
- 단순 리베이트 사이트와 달리 Perpdex 비교 툴, 에어드랍 계산기 등 리서치 도구 제공
- 단일 대시보드에서 여러 거래소 리베이트 통합 관리
- 커뮤니티 이벤트(Perp-DEX Day)를 통한 오프라인 신뢰 구축
- 트레이딩 컴피티션을 통한 커뮤니티 인게이지먼트

## 마케팅 목표
- 거래소 연결 완료율 향상 (UID 연결까지)
- 연결 후 첫 정산 확인까지 이탈 방지
- 레퍼럴을 통한 신규 유저 유입 확대
- 플랫폼 신뢰도 구축 (정산 투명성 강조)
- Perp DEX 트레이더 세그먼트 신규 확보

## 현재 UX 문제
- 사용자가 페이백 구조를 직관적으로 이해하지 못함
- 거래소 연결 후 실제로 혜택이 발생하고 있는지 체감이 안 됨
- 정산 시점, 정산 금액, 출금 가능 시점이 불명확하게 느껴짐
- 신규 사용자가 온보딩 단계에서 이탈하는 경우가 많음
- KYC 변경이나 레퍼럴 업데이트 안내가 불충분함
- ReboundX 링크 없이 가입한 사용자는 UID 연결 자체가 불가 → 사전 안내 부족

## 콘텐츠 원칙
- 수익을 과장하지 않음
- 거래를 유도하지 않음 (투자 권유 아님)
- 구조를 보여줌. 설명하지 않음
- 사용자가 직접 계산하지 않아도 되도록 수치를 먼저 노출
- 신뢰를 먼저, 행동 유도는 그 다음
