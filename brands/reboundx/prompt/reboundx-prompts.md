## 04. 온보딩 플로우 카피 작성

```
brand-brief.md를 읽고 아래 온보딩 단계별 UX 카피를 작성해줘.

대상 사용자: 크립토 입문자 (페이백 구조를 처음 접하는 사람)
원칙:
- 사용자가 계산하지 않아도 되도록 수치를 먼저 노출
- 신뢰를 먼저, 행동 유도는 그 다음
- 수익 보장 표현 금지
- 짧고 명확하게. 전문 용어 최소화

작성할 단계:
1. 랜딩 페이지 히어로 섹션 (헤드라인 + 서브텍스트)
2. 거래소 연결 안내 화면 (타이틀 + 설명 + CTA 버튼 텍스트)
3. 연결 완료 후 첫 화면 (상태 메시지 + 다음 액션 안내)
4. 첫 정산 발생 알림 메시지
5. 출금 가능 안내 메시지

각 단계마다 카피 2개 옵션 작성.
결과물을 outputs/onboarding-copy.md로 저장해줘.
```

   (정산 예정 금액 / 정산 완료 금액 / 출금 가능 금액 상태별 레이블)

분석 후:
- 이탈 원인 추정 (UX 문제 / 메시지 문제 / 신뢰 문제 중 분류)
- 각 이탈 구간별 개선 제안 (카피 수정 / 화면 구조 개선 / 추가 안내 메시지)

raw 데이터는 research/funnel-[날짜].md에 저장하고
분석 결과는 outputs/funnel-insight-[날짜].md로 저장해줘.

## 10. 월간 마케팅 성과 리포트 자동 생성

```
아래 파일들을 모두 읽어줘:
- brand-brief.md
- research/ 폴더 안의 모든 ga4 데이터 파일
- outputs/ 폴더 안의 이번 달 전략 및 실행 파일

읽은 내용을 바탕으로 이번 달 마케팅 성과 리포트를 작성해줘.

포함할 내용:
1. 이번 달 목표 대비 실적 요약
   - 채널별 유입량
   - 전환 이벤트 달성 현황 (거래소 연결 / 첫 정산 확인 / 레퍼럴)
   - 미국 vs 동남아 지역별 비교

2. 잘 작동한 것 (채널 / 카피 유형 / 캠페인)

3. 작동하지 않은 것과 원인 추정

4. 다음 달 집중할 포인트 3가지

5. 다음 달 첫 2주 실행 제안

형식: 팀 공유용 리포트 형식으로 작성. 표 활용.
결과물을 outputs/monthly-report-[년월].md로 저장해줘.
```

## 사용 팁

**날짜 자동 입력이 필요하면 Claude Code에 이렇게 말해:**
```
오늘 날짜를 파일명에 자동으로 넣어줘
```

**특정 프롬프트를 반복 실행하고 싶으면:**
```bash
# 매주 월요일 실행용 alias 등록
echo "alias mkt-weekly='cd ~/Desktop/MKT/brands/reboundx && claude'" >> ~/.bash_profile
source ~/.bash_profile
```

**outputs 폴더 정리:**
```bash
ls ~/Desktop/MKT/brands/reboundx/outputs/
```
